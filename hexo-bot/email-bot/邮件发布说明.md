# Hexo 邮件发布机器人

通过 QQ 邮箱发送文章，自动发布到 Hexo 博客。

## 效果演示

给 `9187541@qq.com` 发一封邮件 → 自动创建文章 → 自动回执通知

## 系统架构

```
手机 QQ 邮箱 / 任意邮箱
       ↓ 发邮件到 9187541@qq.com
Windows 定时任务（每分钟）
       ↓
node email-to-hexo.js
       ↓ 解析邮件，写入文章
D:/hexo/source/_posts/*.md
       ↓
GitHub Pages → snowhoo.net
```

## 使用方法

### 发邮件格式

**发件人**：任意邮箱（推荐用 QQ 邮箱手机 App）
**收件人**：`9187541@qq.com`

#### 示例 1：普通文章
```
主题: # 我的第一篇博客文章

今天天气真好，来写点什么。
#日常 #随笔
[cat:生活]
```

#### 示例 2：存入草稿
```
主题: (随便写)
正文:
这是一篇草稿文章。
#草稿
```

#### 示例 3：只用正文写标题
```
主题: (留空或随便写)

# 这篇文章的标题在这里
正文内容...
#技术
```

## 消息格式

| 格式 | 说明 |
|------|------|
| `主题: # 标题` | 用邮件主题作为文章标题 |
| `# 标题`（正文首行） | 用正文第一行作为标题 |
| `#标签` | 自动识别为标签 |
| `[cat:分类名]` | 分类 |
| `[草稿]` | 存入草稿，不发布 |

## 管理命令（通过邮件）

暂无，通过直接编辑 `D:\hexo\source\_drafts` 管理草稿。

## 服务状态

- **定时任务**：`HexoEmailBot`（每 1 分钟执行一次）
- **查看运行日志**：任务计划程序 → HexoEmailBot → 最近运行结果
- **停止服务**：`schtasks /end /tn HexoEmailBot`
- **卸载服务**：`schtasks /delete /tn HexoEmailBot /f`

## 常见问题

**Q: 邮件发出后没反应？**
定时任务每 1 分钟轮询一次，最长等待 1 分钟才会处理。确认发件邮箱已发出，收件人是 `9187541@qq.com`。

**Q: 收到回执但文章没发布？**
检查 `D:\hexo\source\_posts` 是否有对应 md 文件。如果文件名含 `[草稿]` 会被存到 `_drafts` 而不是 `_posts`。

**Q: 想改成自动 deploy？**
编辑 `D:\hexo\hexo-bot\email-to-hexo.js`，把 `AUTO_DEPLOY = false` 改成 `AUTO_DEPLOY = true`。
